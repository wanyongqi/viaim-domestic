# 典型缺陷库

从真实缺陷中提炼的**通用缺陷模式**，适用于任何功能模块的测试点设计。每个模式包含通用规律、真实案例和跨模块映射。

> 实证数据：V2.0.2 真实缺陷 10 条（设备绑定与未激活会员模块）
> 最后更新：2026-04

## 模式A：外部/异步操作后的状态同步

### 通用规律

**当系统的状态可以被"当前操作界面之外"的途径改变时，当前界面需要感知并同步这些变更。**

适用场景：设备端独立操作、多端同步、后台任务完成、服务端推送状态变更。

### 检查清单（适用于任何模块）

- [ ] 外部途径改变了状态 → 当前界面是否感知并刷新
- [ ] 外部操作进行中 → 当前界面连接/打开时是否同步进行中状态
- [ ] 外部操作已完成 → 当前界面是否展示最终结果

### 实证案例（V2.0.2 缺陷1）

> 耳机未连接App时独立开启闪录 → 启动App连接耳机 → 底部没有显示闪录状态

### 跨模块映射

| 模块 | 同类场景 |
|------|---------|
| 多端同步 | Web 端删除了记录 → App 端列表未刷新 |
| 后台任务 | 上传/转写在后台完成 → 返回详情页未展示结果 |
| 订阅 | 其他端完成购买/取消 → 当前端套餐状态未更新 |
| 区分说话人 | 区分进行中离开详情页 → 返回后未展示区分结果 |

## 模式B：前后端上线时序的兼容窗口

### 通用规律

**当功能依赖前后端协同时，存在"服务端已上线新逻辑、客户端尚未升级"的灰度窗口期，新旧逻辑可能冲突。**

适用场景：任何需要服务端配合的新功能、接口变更、规则变更。

### 检查清单（适用于任何模块）

- [ ] 服务端新逻辑 + 老版本客户端 → 是否正常降级/兼容
- [ ] 老版本注册用户 + 新版本客户端 → 数据迁移是否正确
- [ ] 服务端灰度阶段 → 部分用户命中新逻辑、部分未命中时行为是否一致

### 实证案例（V2.0.2 缺陷2）

> 服务端上线"未激活会员"逻辑后，激活条件设为「必须完成设备绑定」。但老版本App没有绑定流程（V2.0.2才新增），导致老版本注册且连接过耳机的用户升级到V2.0.2后被误判为未激活会员，无法上传。修正：老版本连接过耳机的用户直接视为已激活 Basic。
>
> **根因**：服务端新逻辑未考虑老版本客户端不存在绑定流程这一前提。

### 跨模块映射

| 模块 | 同类场景 |
|------|---------|
| 订阅/账单 | 新增账单页功能 → 老版本App无此入口，服务端接口是否兼容 |
| 兑换码 | 兑换码新规则上线 → 老版本App的兑换流程是否正常 |
| AI功能 | AI 模型权限变更 → 老版本对新模型的请求处理 |

## 模式C：实体属性差异导致通用流程的分支遗漏

### 通用规律

**当同一流程需要处理多种"实体类型"时，容易按主流类型实现，遗漏非主流类型的差异化处理。**

适用场景：不同设备型号、不同文件类型、不同套餐等级、不同支付渠道、不同用户角色。

### 检查清单（适用于任何模块）

- [ ] 列出该流程涉及的所有实体类型（设备型号/文件格式/套餐/渠道等）
- [ ] 不支持该功能的实体类型 → 是否正确跳过/隐藏相关UI
- [ ] 不同实体类型的同一校验逻辑 → 实现是否有差异
- [ ] 不属于当前适用范围的实体 → 入口层是否拦截并正确引导

### 实证案例（V2.0.2 缺陷3/7/8，共3条）

> **缺陷3**：国内耳机连接海外App → 弹出绑定弹窗（应弹国内引导弹窗）
> **缺陷7**：Air单耳在线 → 弹出绑定弹窗（应弹双耳在线弹窗，Air的单耳检测逻辑不同）
> **缺陷8**：解绑Air耳机 → 弹窗提示清除闪录（Air不支持闪录，不应提示）

### 跨模块映射

| 模块 | 实体类型差异 | 易错点 |
|------|------------|--------|
| 文件分享 | 音频 vs 文档 | 文档不消耗转写时长、文档必须选Space、文档不在记录列表展示 |
| 账单管理 | viaim/Apple/Google/兑换码 | 不同支付渠道的详情弹窗内容不同、兑换码无金额无发票按钮 |
| 订阅兑换码 | Basic/Pro/Ultra 用户 | 有活跃订阅需二次确认、无订阅直接兑换 |
| 区分说话人 | 双路通话/单路/现场/同传 | 取消区分后双路回退到"我方/对方"，其他隐藏标签 |
| 上传 | 不同音频格式/大小/时长 | 校验优先级（时长>大小>格式） |

### 实体属性差异矩阵模板

遇到此类场景时，建立如下矩阵后逐格验证：

```
           | 实体A | 实体B | 实体C |
|----------|-------|-------|-------|
| 功能/步骤1 |  ✅   |  ❌   |  ✅   |
| 功能/步骤2 |  ✅   |  ✅   |  ❌   |
| 功能/步骤3 |  ✅   |  ❌   |  ❌   |
```

## 模式D：阈值/版本号等边界比较逻辑错误

### 通用规律

**涉及数值比较、版本号比较、时长/大小限制等阈值判断时，边界值处的实现容易出错（尤其是多段版本号的字符串比较）。**

适用场景：固件版本号、App版本号、文件大小限制、时长限制、字数限制、数量上限。

### 检查清单（适用于任何模块）

- [ ] 刚好等于阈值 → 属于哪一侧（含/不含）
- [ ] 刚好超过/不足阈值 1 个最小单位 → 行为是否正确
- [ ] 多段数值（如版本号 5.1.0.9 vs 5.1.0.32）→ 是否按数值而非字符串比较
- [ ] 主段跨越（如 4.9.9.99 vs 5.0.0.1）→ 不会因子版本号大而误判

### 实证案例（V2.0.2 缺陷4）

> 低固件Pro3/Air2（不达标版本）首次连接后误弹录音提示音说明页。测试点已覆盖，属开发实现错误。

### 跨模块映射

| 模块 | 同类阈值 | 边界测试数据 |
|------|---------|------------|
| 文件分享 | 文件大小 200MB | 199.9MB / 200MB / 200.1MB |
| 文件分享 | 音频时长 1小时(Basic) / 4小时(Pro) | 59:59 / 1:00:00 / 1:00:01 |
| 兑换码 | 输入长度 ≤7位置灰 / >7位高亮 / 30位上限 | 7位 / 8位 / 30位 / 31位 |
| 摘要模板 | 提示词 2000 字限制 | 1999字 / 2000字 / 2001字 |
| Space | 数量上限 10(Basic)/20(Pro)/40(Ultra) | 刚好达上限 / 超出1个 |
| 上传队列 | 队列满 6 个 | 5个+1 / 6个+1 |

## 模式E：多入口触发同一流程时中间步骤被跳过

### 通用规律

**同一核心流程有多个触发入口时，各入口的前置状态不同，可能导致某些中间步骤在特定入口下被意外跳过。**

适用场景：任何有多个入口的功能（页面跳转入口、弹窗内链接、深链、外部分享等）。

### 检查清单（适用于任何模块）

- [ ] 列出所有入口（页面按钮/弹窗链接/快捷方式/外部调起等）
- [ ] 核心流程的每个关键中间步骤 → 逐入口验证是否正确触发
- [ ] 非常规入口（自动触发/外部调起/深链）→ 中间步骤是否完整

### 实证案例（V2.0.2 缺陷5）

> 从耳机设置页触发连接Air2 → MFI弹窗未弹出（从首页入口连接时正常弹出）

### 跨模块映射

| 模块 | 多入口场景 | 需逐入口验证的中间步骤 |
|------|----------|---------------------|
| 文件分享 | 系统分享/文件管理器/其他App | 文件校验→配置弹窗→空间选择→加入队列 |
| 上传 | Record页添加/Space页添加/分享入口 | 格式校验→时长校验→转写时长校验→配置弹窗 |
| 解锁Basic | Record添加记录/Space添加来源/我的页面按钮 | 弹窗内容→添加设备→购买权益链接 |
| 订阅 | 账单页/我的页面/升级弹窗/兑换码成功后 | 套餐选择→支付→权益生效→多端同步 |
| 删除 | 详情页更多/列表左滑/批量选择 | 二次确认弹窗→删除执行→列表刷新→空间数量更新 |

## 模式F：操作失败/取消后状态未正确回退

### 通用规律

**测试通常聚焦"操作成功"后的状态变更，而遗漏"操作失败"或"用户中途取消"后的状态回退和资源清理。**

适用场景：任何会改变UI状态、消耗资源、启动后台进程的操作。

### 检查清单（适用于任何模块）

对每个"会改变状态的操作"，成对验证：
- [ ] 操作成功 → 状态正确更新 ✅（通常已覆盖）
- [ ] **操作失败（网络/服务端/校验）→ 状态不更新/回退到操作前** ❗（高频遗漏）
- [ ] **用户主动取消/中途退出 → 状态回退 + 后台进程停止 + 资源返还** ❗（高频遗漏）

### 实证案例（V2.0.2 缺陷6/9，共2条）

> **缺陷6**：连接失败 → 首页图标误更新为失败设备（应仅在连接成功后更新）
> **缺陷9**：搜索中手动退出 → 耳机设置页仍显示"搜索中"（应停止搜索并清除状态）

### 跨模块映射

| 模块 | 操作 | 失败/取消后需验证的回退 |
|------|------|---------------------|
| 文件分享 | 上传 | 取消→转写时长返还、Space数量返还、队列清除 |
| 订阅 | 取消/恢复订阅 | 请求失败→按钮状态不变、卡片信息不变 |
| 账单 | 申请发票 | 提交失败→输入框内容保留、按钮可重试 |
| 兑换码 | 兑换 | 校验失败→输入内容保留、错误提示展示、按钮可重新点击 |
| 区分说话人 | 区分 | 区分失败→按钮恢复到操作前状态（首次→"识别说话人"，已区分3人重试→"已区分3个"） |

## 模式G：多实体并存/叠加时的状态判定异常

### 通用规律

**当系统中同时存在多个同类实体（多设备、多套餐来源、多上传任务）时，切换或叠加场景下的状态判定容易出错。**

适用场景：多设备连接、套餐叠加（付费+兑换码）、多文件上传、多Space操作。

### 检查清单（适用于任何模块）

- [ ] 多个同类实体并存时 → 状态判定依据是否明确且唯一
- [ ] 实体之间切换 → 中间态的状态是否正确
- [ ] 实体叠加 → 优先级/展示/到期回落逻辑是否正确
- [ ] 其中一个实体状态变更 → 是否影响其他实体

### 实证案例（V2.0.2 缺陷10）

> 经典蓝牙同时连接Pro2和Air2 → 断开Pro2后通过添加设备连接Air2 → 已连接过的Air2仍弹出欢迎弹窗

### 跨模块映射

| 模块 | 多实体并存场景 | 易错点 |
|------|-------------|--------|
| 订阅兑换码 | Pro付费+Ultra兑换码并存 | 展示最高等级、到期后正确回落、权益取并集还是最高级 |
| 文件分享 | 文档选择多个Space上传 | 每个Space独立计数、其中一个满不影响其他 |
| 上传队列 | 分享上传+耳机上云同时进行 | 两者独立队列互不影响 |
| 区分说话人 | 多设备互斥操作 | A区分中B点击→Toast不可操作、A完成后B可操作 |

## 高频遗漏维度排名

| 排名 | 通用遗漏维度 | 实证缺陷数 | 一句话检查 |
|------|------------|-----------|----------|
| 1 | **实体属性差异的分支遗漏**（模式C） | 3 | 不同类型的实体走同一流程时，差异化处理是否到位？ |
| 2 | **操作失败/取消后状态回退**（模式F） | 2 | 除了成功路径，失败和取消后的状态是否正确回退？ |
| 3 | **多入口中间步骤验证**（模式E） | 1 | 每个入口进入的流程，中间步骤是否都完整执行？ |
| 4 | **外部操作后的状态同步**（模式A） | 1 | 其他端/设备端/后台改变了状态后，当前界面是否感知？ |
| 5 | **前后端上线时序兼容**（模式B） | 1 | 服务端先上线、客户端未升级的窗口期是否正常？ |
| 6 | **多实体并存时的判定**（模式G） | 1 | 多个同类实体同时存在时，状态判定和切换是否正确？ |
| 7 | **阈值边界比较**（模式D） | 1 | 版本号/数量/时长等边界值的比较逻辑是否正确？ |

## AI 使用指南

### 生成测试点时

1. **必读本文件**，对照"高频遗漏维度排名"中的 7 个维度逐一检查新需求
2. **不要局限于连接模块**——每个模式下的"跨模块映射"表展示了该模式在其他模块中的具体表现
3. 遇到具体场景时，使用对应模式的"检查清单"逐项验证

### 判断适用模式的快速索引

| 需求特征 | 优先检查模式 |
|---------|------------|
| 涉及多种实体类型（文件格式/套餐/渠道/设备型号） | **模式C** |
| 涉及状态变更操作（提交/删除/连接/上传） | **模式F** |
| 同一功能有多个触发入口 | **模式E** |
| 涉及数值阈值/版本号判断 | **模式D** |
| 涉及多端/设备端/后台操作 | **模式A** |
| 涉及前后端协同或版本升级 | **模式B** |
| 涉及多个同类实体并存或叠加 | **模式G** |
