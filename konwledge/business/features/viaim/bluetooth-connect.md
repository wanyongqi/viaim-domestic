# 耳机连接流程规格（viaim）

> 版本：V2.0.2 | 最后更新：2026-04  
> 耳机型号与设置功能详见 `features/viaim/bluetooth-headset.md`

---

## 一、首次连接流程

### 1.1 权限要求（按平台）

| 平台 | 所需权限/服务 |
|------|-------------|
| iOS | 蓝牙权限（Bluetooth）|
| Android 12+ | 附近的设备权限（Nearby Devices）|
| Android 12- | 位置信息权限（Location）+ 定位服务已开启 |

### 1.2 权限引导规则

**iOS：**

| 权限状态 | 触发行为 |
|---------|---------|
| 未获取蓝牙权限，点击添加设备 | 弹出系统蓝牙授权弹窗 |
| 首次点击不允许 | **不弹**「去设置」弹窗 |
| 非首次拒绝/关闭后 | 弹出「去设置」弹窗；点击「去设置」跳转系统设置；点击取消/关闭回到记录页 |
| 已授权 + 蓝牙已开启 | 直接进入搜索设备页面 |
| 已授权 + 蓝牙未开启 | 拉起「请打开蓝牙」页面；开启蓝牙后点击继续 → 跳转搜索设备页 |

**Android 12+：**

| 权限状态 | 触发行为 |
|---------|---------|
| 未获取附近设备权限，点击添加设备 | 弹出「需要附近的设备」弹窗 |
| 点击「去开启权限」| 弹出系统授权选项 |
| 点击拒绝/关闭 | 回到记录页 |
| 点击「去设置」 | 跳转系统设置 |
| 已授权 + 蓝牙已开启 | 进入搜索设备页面 |
| 已授权 + 蓝牙未开启 | 拉起「请打开蓝牙」页面 |

**Android 12-：**

| 权限状态 | 触发行为 |
|---------|---------|
| 未获取位置权限，点击添加设备 | 弹出「需要位置信息权限」弹窗 |
| 点击「去开启权限」| 弹出系统授权选项 |
| 拒绝/关闭 → 弹出「去设置」弹窗 | 点击「去设置」跳转系统设置；点击取消/关闭回到记录页 |
| 已授权位置权限 + 定位服务未开启 | 弹出「需要开启定位服务」弹窗；点击「去设置」跳转系统设置 |
| 已授权 + 定位已开启 + 蓝牙已开启 | 进入搜索设备中页面 |
| 已授权 + 定位已开启 + 蓝牙未开启 | 拉起「请打开蓝牙」页面 |

> 各平台：曾经拒绝权限后退出 App 重新进入，再次点击添加设备仍会弹出对应权限弹窗。

### 1.3 气泡引导

| 场景 | 行为 |
|------|------|
| 首次登录 App | 记录列表页左上角添加设备按钮旁展示气泡引导「在这里添加你的 viaim 设备」|
| 点击添加设备按钮 | 气泡引导消失 |
| 退出后再次登录 | **不再显示**（仅首次登录才显示）|
| 卸载重装后 | 重新显示气泡引导 |

### 1.4 搜索设备流程

```
点击添加设备 → 进入搜索设备中页面
  ├─ 搜索中：左上角返回 → 记录页；底部「手动添加」→ 未发现设备页
  ├─ 系统蓝牙已连接 2 个耳机 → 优先连接最后连接系统的耳机
  ├─ 搜索中关闭系统蓝牙 → 搜索超时
  ├─ 长时间未搜到 → 搜索超时页
  └─ Android：搜索到设备 → 弹出连接中弹窗
```

**搜索超时页：**
- 底部显示「重新扫描」和「手动添加」按钮
- 点击「重新扫描」→ 再次进入搜索中；点击「手动添加」→ 未发现设备页；左上角返回 → 记录页

**未发现设备页：**
- 点击各型号耳机（air / recdot / recdot2 / opennote / nano+）→ 弹出该型号「如何连接」引导弹窗
- 弹窗内：点击关闭/弹窗外/选择产品 → 关闭；点击「前往设置」→ 系统蓝牙设置
- 左上角返回 → 回到搜索超时页

### 1.5 连接中与失败处理

| 场景 | 行为 |
|------|------|
| 连接中切换 App 到后台再切回 | 连接状态保持，继续连接 |
| 连接中系统蓝牙断连 | 弹出「无法连接到」弹窗 |
| 弹窗内点击「重新连接」| 再次进入连接中弹窗 |
| 累计重连 3 次仍失败 | 弹窗显示「重置蓝牙配对」按钮 → 点击弹出重置操作引导弹窗 |
| 重置引导弹窗点击关闭 | 关闭弹窗 |

### 1.6 首次连接成功（V2.0.2变更）

- 记录页顶部显示连接耳机动画
- **欢迎弹窗按设备维度触发**：本地以耳机SN为key记录是否已弹出过，每个物理耳机首次连接均弹出
  - 连接不同型号（如Pro3→Air2）→ 分别弹出
  - 同型号更换新耳机（不同SN）→ 新耳机也弹出
  - 卸载重装App → 本地记录清空，所有耳机重新触发
- 弹出「欢迎使用」弹窗（图片与实际连接型号一致）
- 点击弹窗外 → **不关闭弹窗**
- 点击「开始使用」→ 进入耳机更多设置页面，同时根据条件触发后续引导：
  1. **录音提示音说明页**（V2.0.2新增）
  2. **耳塞贴合度检测**
- **非首次连接**（本地已有该耳机弹窗记录）：不弹欢迎弹窗，仅toast提示

#### 录音提示音说明页（V2.0.2新增）

**触发条件矩阵：**

| 型号 | 区域 | 最低固件版本 | 开关状态 | 是否弹出 |
|------|------|------------|---------|---------|
| Pro3 | 欧美 | ≥ 5.1.0.32 | 开启 | ✅ |
| Pro2 | 欧美 | ≥ 5.0.6.10 | 开启 | ✅ |
| Air2 | 欧美 | ≥ 7.0.0.70 | 开启 | ✅ |
| Pro3/Pro2/Air2 | **亚太** | 任意 | 任意 | ❌ |
| Nano+ / VIAIM Air | — | — | 任意 | ❌ |
| 欧美耳机 | 欧美 | **低于**对应阈值 | 任意 | ❌ |
| 欧美耳机 | 欧美 | 达标 | **关闭** | ❌ |
| 非首次连接 | — | — | 任意 | ❌ |

**页面规格：** 全屏页面，展示录音提示音开关（默认开启）和确认按钮。

**页面交互：**

| 操作 | 行为 |
|------|------|
| 开关开启 → 点击确认 | 关闭说明页，提示音保持开启，进入后续流程 |
| 关闭开关 | 展开责任声明勾选项，确认按钮置灰不可点击 |
| 关闭开关 → 勾选责任声明 | 确认按钮恢复可点击 |
| 关闭开关 → 勾选声明 → 点击确认 | 提示音变为关闭，进入后续流程 |
| 关闭开关 → 再切回开启 | 责任声明收起隐藏，确认按钮恢复可点击 |

**后续状态规则：**
- 首次连接弹出说明页后 → 后续手动开关提示音不再弹出说明页
- 卸载重装App → 首次连接欧美耳机重新弹出说明页

#### 耳塞贴合度检测（首次连接弹窗）

| 型号 | 行为 |
|------|------|
| 欧美 Pro3/Pro2 | 说明页关闭后 → 弹出贴合度检测弹窗 |
| 亚太 Pro3/Pro2/Nano+ | 无说明页 → 欢迎弹窗后直接弹出 |
| Air2 / VIAIM Air | 不弹（开放式耳机）|
| 非首次连接 | 不弹 |

- 说明页中关闭提示音后 → 仍弹出贴合度检测（两者独立）
- 卸载重装后首次连接入耳式耳机 → 重新弹出

## 二、非首次连接与自动重连

### 2.1 App 打开时自动重连

- 前提：曾经连接过耳机 + 所有权限已授权 + 系统蓝牙已开启
- 首页左上角显示「正在连接」（iOS 显示「连接中」）

**权限/服务不足时的自动重连：**

| 条件 | 行为 |
|------|------|
| iOS 无蓝牙权限 | 直接弹出系统蓝牙授权弹窗 |
| iOS / Android 系统蓝牙未开启 | 直接拉起「请打开蓝牙」页面 |
| Android 12+ 无附近设备权限 | 直接弹出需要附近设备权限弹窗 |
| Android 12- 无位置权限 | 弹出需要位置信息权限弹窗 |
| Android 12- 已授权位置权限 + 定位未开启 | 直接弹出需要开启定位服务弹窗 |
| 权限/服务弹窗被拒绝/取消 | 首页状态显示「未连接」|

### 2.2 重连结果

| 结果 | 首页表现 |
|------|---------|
| 搜索到设备 + 连接成功 | 显示耳机连接成功动画 |
| 搜索到设备 + 连接失败 | 状态变为「未连接」|
| 搜索超时（未搜到设备）| 状态变为「未连接」|
| 上次连接耳机 A + 系统蓝牙已连接耳机 B | 连接耳机 B 成功 |

> iOS：蓝牙无连接设备时进入 App 直接显示「未连接」，不触发搜索。

### 2.3 首页连接状态交互

| 点击状态 | 行为 |
|---------|------|
| 点击「正在搜索/连接中」| 进入耳机设置缺省页，继续搜索，显示正在搜索 |
| 点击「未连接」| 进入耳机设置缺省页，触发自动重连，显示正在搜索 |

### 2.4 耳机设置页中的重连

```
进入耳机设置页 → 自动触发重连搜索
  ├─ 搜索到设备 + 连接成功 → 页面更新显示设置项，状态显示已连接
  ├─ 搜索到设备 + 连接失败 → 弹出「无法连接到[设备名]」弹窗
  │     Android：显示当前连接耳机弹窗；iOS：显示上次连接设备弹窗
  ├─ 重连搜索超时（未搜到）→ 弹出上次连接耳机的「如何连接」引导弹窗
  │     点击「去设置」→ 系统蓝牙设置；点击「选择其他产品」→ 未发现设备页
  │     返回记录页 → 右上角显示「未连接」
  └─ 重连失败 → 连接状态区域显示「重新连接」按钮
        点击「重新连接」→ 状态变「正在搜索...」，再次发起重连
        累计 3 次重连失败 → 显示「重置蓝牙配对」按钮 → 点击弹出重置指引
```

进入耳机设置页时若权限不足，触发行为与 App 打开时自动重连相同。

### 2.5 耳机设置页添加设备按钮（V2.0.2 新增）

- 耳机设置页右上角新增「添加设备」按钮
- 点击进入搜索设备页面，走搜索绑定流程（见五、设备绑定流程）

| 当前状态 | 点击添加设备 |
|---------|------------|
| 未连接 | 进入搜索设备页面 |
| 已连接 | 弹「请先断开」弹窗（支持 X / 下拉 / 弹窗外关闭）|
| 搜索中 | 按钮置灰，点击无响应 |

### 2.6 断开连接的即时响应

- 当前在设备详情页（耳机设置页）时耳机关盖/断开 → 页面状态**即时**更新为「重新连接」，所有设置项变为缺省状态

### 2.7 未连接状态下的耳机设置页

- 进入耳机设置更多页时（点击「正在搜索/连接中」或「未连接」进入）→ 显示**上次连接耳机的图片**，设置项为缺省状态

## 三、MFI 连接流程（Air2/opennote 专属）

> Air2（opennote）使用 MFI（Made for iPhone）协议，首次连接时走专属的配对发现页面，而非普通蓝牙扫描流程。

### 3.1 首次添加 opennote 设备

| 耳机状态 | 行为 |
|---------|------|
| 未开盖 | 搜索超时，无法搜索到 |
| 开盖但**未进入配对状态** | 搜索到 opennote，进入「发现 opennote 设备」页面；点击配对 → 配对列表**为空** |
| 开盖 + **长按进入配对状态** | 点击配对后列表显示耳机名称，可选择连接 |
| 开盖并**已进入配对状态**（长按）| 进入「发现 opennote 设备」页面，弹出配对列表，直接显示耳机名称 |
| 多个 opennote 同时处于配对状态 | 配对列表显示所有耳机名称，供用户选择 |

**MFI 发现页超时规则：**
- 发现页有 15 秒超时限制
- 超时后点击返回 → 回到搜索超时页；点击重新扫描 → 进入搜索中；点击手动添加 → 未发现设备页

### 3.2 MFI 连接失败处理

| 操作 | 行为 |
|------|------|
| 配对列表点击耳机名称 + 连接失败 | 弹出「无法连接到 [上次连接耳机名称]」弹窗 |
| 弹窗点击关闭/弹窗外 | 关闭弹窗 |
| 弹窗点击「重新连接」| 再次发起连接流程 |
| 重连成功 | 连接成功 |
| 累计 3 次重连失败 | 弹窗显示「重置蓝牙配对」按钮 → 点击弹出重置指引弹窗 |
| 关闭重置指引弹窗后再次重连失败 | **不显示**「重置蓝牙配对」按钮（需重新累计失败 3 次）|

### 3.3 MFI 自动重连

| 场景 | 结果 |
|------|------|
| 之前连接过 opennote + 开盖 + 打开 App | 自动重连成功 |
| 之前连接过 opennote + 开盖进入配对 + 耳机设置页 | 自动重连成功 |
| 之前连接过其他耳机（如 recdot2）+ opennote 开盖 + 耳机设置页 | 自动重连至 opennote 成功 |
| 之前连接过 opennote1 + opennote2 开盖进配对 + 耳机设置页 | 可连接 opennote2（若系统支持切换连接）|

### 3.4 系统蓝牙直连 opennote 时的 MFI 弹窗（iOS）

- 通过系统蓝牙首次连接 opennote → 连接成功后弹出「viaim 请求通信」弹窗
- 点击「允许」→ 拉起 App 并连接成功
- 点击「取消」→ **不会拉起 App**

## 四、国际版 App 连接国内耳机规则

> 国际版 App 对国内版耳机有白名单限制。

### 4.1 白名单型号（可正常连接）

以下国内版耳机连接国际版 App 后，功能和 UI 正常：

| 国内型号 | 对应国际型号 |
|---------|------------|
| Air | VIAIM Air |
| Nano+ | Nano+ |
| Pro2 | recdot |
| Pro3 | recdot2 |
| Air2 | opennote |

### 4.2 非白名单型号（不支持连接）

国内版 nano/pro/iFLYBUDS/iFLYBUDS2 等**非白名单型号**连接国际版 App 时：

1. 连接成功后**立即自动断开**
2. 弹出「不支持连接」弹窗，提示下载国内版 App
3. 点击「关闭」即可关闭弹窗

**弹窗内操作（查看安装方式）：**

| 平台 | 行为 |
|------|------|
| iOS | 弹出「安装方式」引导弹窗：「在 App Store 中查看」→ TestFlight 下载；「打开邀请链接」→ 国内 IP 跳国内版 / 海外 IP 跳国外版；「联系客服」→ 客服聊天页 |
| Android | 跳转手机浏览器打开官网国内版 App 下载页 |

### 4.3 封禁耳机规则

当耳机本身被封禁（与型号无关，任意国内型号均可被封禁）时：

- 连接并打开 App 后，显示**封禁页面**（具体 UI 以设计稿为准）
- 涉及型号：Air / Nano+ / Air2 / Pro2 / Pro3 / nano / pro / iFLYBUDS / iFLYBUDS2

## 五、设备绑定流程（V2.0.2 新增）

> V2.0.2 起，搜索到设备后需经过绑定判定，未绑定设备需先绑定才能使用。

### 5.1 搜索时绑定判定矩阵

| 网络状态 | 缓存/绑定状态 | 行为 |
|---------|-------------|------|
| 正常 | 已绑定本账户设备 | 直接连接成功 |
| 正常 | 设备未绑定 + 双耳在线 | 弹出绑定弹窗 |
| 正常 | 设备已被其他账号绑定 | 弹出已被绑定弹窗，停止搜索 |
| 正常 | 设备未绑定 + 一只耳机未在线 | 弹出需要连接两只耳机弹窗，停止搜索 |
| 异常 | 有缓存 + 已绑定此账号 | 直接连接成功（离线降级）|
| 异常 | 有缓存 + 未绑定此账号 | 弹出网络不佳弹窗，停止搜索 |
| 异常 | 无缓存 | 弹出网络不佳弹窗 |

### 5.2 绑定弹窗

**内容：** 设备图片 + 绑定文案 + 绑定按钮

| 操作 | 行为 |
|------|------|
| 点击绑定 | 显示绑定中 loading |
| X / 顶部下拉关闭 | App 未连接状态 |
| 点击弹窗外 | **不支持关闭** |
| 绑定中状态 | 不可关闭弹窗，不可重复点击绑定按钮 |

**绑定结果：**

| 结果 | 行为 |
|------|------|
| 绑定成功 + 首次连接该耳机 | App 显示已连接，弹出欢迎弹窗，走连接成功后弹窗流程 |
| 绑定成功 + 非首次连接该耳机 | Toast「设备绑定成功」|
| 绑定中网络异常 | 覆盖弹出网络不佳弹窗，App 保持连接中动画 |

### 5.3 异常弹窗

**已被绑定弹窗：**

| 操作 | 行为 |
|------|------|
| 知道了 / X / 下拉关闭 | 关闭弹窗 |
| 点击弹窗外 | **不可关闭** |
| 点击切换账号 | 跳转账号设置页 |

**需要连接两只耳机弹窗：**

| 操作 | 行为 |
|------|------|
| 点击重试 | 重新搜索 |
| X / 下拉关闭 | 关闭弹窗 |
| 点击弹窗外 | **不可关闭** |

**网络不佳弹窗（绑定中/搜索中通用）：**

| 操作 | 行为 |
|------|------|

## 六、测试关注点

**首次连接：**
- iOS 首次拒绝蓝牙权限：**不弹**「去设置」弹窗（非首次拒绝才弹）
- Android 12- 需位置权限 + 定位服务双重检查
- 搜索中有 2 个系统蓝牙耳机时：优先连接最后连接系统的耳机
- 重连 3 次失败才显示「重置蓝牙配对」按钮
- 欢迎使用弹窗：**按设备维度**（每个耳机SN独立追踪），点击弹窗外不关闭

**录音提示音说明页（V2.0.2）：**
- 触发条件三维矩阵：型号×地区×固件版本，亚太版/Nano+/Air不弹
- 固件版本阈值：Pro3 ≥ 5.1.0.32、Pro2 ≥ 5.0.6.10、Air2 ≥ 7.0.0.70（注意多段版本号比较）
- 关闭提示音流程：关闭开关→展开声明→置灰按钮→勾选→恢复可点击
- 关闭后再切回开启：声明收起，按钮恢复
- 首次弹出后不再重复弹出；卸载重装后重新弹出
- 说明页与贴合度检测独立（关闭提示音后仍弹贴合度检测）

**设备绑定（V2.0.2）：**
- 搜索绑定判定矩阵：网络×缓存×绑定状态，共7条路径全覆盖
- 绑定中状态防护：不可关闭弹窗、不可重复点击
- 绑定成功区分首次/非首次（欢迎弹窗 vs Toast）
- 4类异常弹窗关闭方式差异：均不支持弹窗外关闭
- 多入口触发（6个入口）：核心流程一致，但入口状态差异需逐入口验证

**非首次连接与复连：**
- iOS 蓝牙无连接设备时进入 App 直接显示「未连接」，不触发搜索
- 耳机设置页搜索超时（未搜到）→ 弹上次连接耳机引导弹窗，而非直接显示未连接
- Android vs iOS 连接失败弹窗内容不同（Android 显示当前，iOS 显示上次）
- 断开时设备详情页设置项**即时**变为缺省状态

**MFI（Air2/opennote）：**
- opennote 必须开盖 + 长按进入配对状态，配对列表才有耳机名称
- 关闭重置蓝牙配对弹窗后，失败计数**重置**，需再次失败 3 次才显示重置按钮
- 通过系统蓝牙直连 opennote：弹 viaim 请求通信弹窗；取消不会拉起 App

**国内耳机与封禁：**
- 国际版 App 白名单国内型号（Air/Nano+/Pro2/Pro3/Air2）可正常连接
- 非白名单国内型号：连接后立即断开 + 弹「不支持连接」弹窗
- iOS「查看安装方式」：区分国内/海外 IP 跳转不同 TestFlight 版本
- 封禁耳机：任意被封禁型号连接后显示封禁页面
