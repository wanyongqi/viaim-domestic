# Space 功能规格（viaim）

> 版本：V2.0.0 | 最后更新：2026-03

---

## 一、Space 概念

Space（空间）是记录的组织容器。可以把记录加入 Space，基于 Space 进行 Vitana 问答、批量操作、文档管理。

**Space 与记录的关系：**
- 一条记录可加入多个 Space
- 加入 Space 的记录以「快捷方式」形式展示（音频）
- 文档直接属于 Space，无记录列表原文件

---

## 二、Space 列表操作

| 操作 | 说明 |
|------|------|
| 新建空间 | 可新建 Space |
| 编辑空间 | 左滑 → 编辑空间名称 |
| 删除空间 | 左滑 → 删除，Space 内所有记录均删除到回收站 |

> 套餐上限：Basic 10 个 / Pro 20 个 / Ultra 40 个，超限行为详见 `subscription.md` 第一节。

---

## 三、Space 内添加记录

| 添加方式 | 说明 |
|----------|------|
| 从记录库添加 | 弹窗选择记录列表中的记录，在 Space 内创建快捷方式 |
| Space 内上传音频 | 上传成功后，在记录列表创建原文件，此 Space 创建快捷方式 |
| Space 内上传文档 | 直接属于该 Space，记录列表不展示 |

**限制：未上云记录不可添加到 Space。**

> 单 Space 文件上限：Basic 10 个 / Pro·Ultra 50 个，超限行为详见 `subscription.md` 第一节。

**容量占用细则：**
- 上传中/等待上传文件**计入**空间已用数量
- 解析失败记录**占用**空间容量（删除后释放）
- 置灰快捷方式（原文件已删到回收站）**依然占用**空间容量；原文件彻底删除后释放
- 上传失败/取消上传 → 空间已用数量返还
- 跨设备：B 设备不包含 A 设备上传中/等待上传文件（客户端本地状态不同步）

> 详细规则见 `upload-rules.md` 第八节"空间容量占用规则"。

## 三-B、Space 详情页（来源 Tab）

### 页面结构

| 区域 | 内容 |
|------|------|
| 顶部 | 空间名称 + 排序按钮 |
| 中部 | 文件列表（快捷方式 + 文档原文件） |
| 底部 Tab | 「来源」（默认展示）+「与 Vitana 聊天」 |
| 空状态 | 中部展示添加来源按钮 + 支持文件类型提示 |

### 添加来源入口

| 入口 | 位置 |
|------|------|
| 空状态 | 中部**添加来源按钮** |
| 有文件状态 | 右下角**添加来源按钮** |

#### 未激活会员限制（V2.0.2 设备绑定需求）

未激活会员（未绑定设备的新用户）点击添加来源按钮 → **弹出解锁 Basic 会员弹窗**，引导用户绑定 viaim 设备激活 Basic 权益。

> 解锁弹窗内容及设备绑定完整规则 → 详见 `features/viaim/device-binding.md`

#### 正常上传流程

弹出添加到空间弹窗，显示两种添加方式：

| 方式 | 说明 |
|------|------|
| 记录库 | 从 Record 列表选择已上云成功的记录，添加为快捷方式 |
| 本地文件 | 从系统文件管理器选择音频/文档文件上传 |

### 记录库弹窗规则

| 规则 | 说明 |
|------|------|
| 展示范围 | 上云成功的四种录音 + 上传并解析成功的音频 + 下载并上云成功的闪录 |
| 置灰范围 | 仅录音记录（无转写）**置灰不可选择**（已上云但无转写内容） |
| 隐藏范围 | 未上云、解析中、解析失败的记录**隐藏不展示**（与批量模式的"置灰可见"不同） |
| 排序 | 按创建时间倒序排列 |
| 选择方式 | 可多选，无选择上限（受空间容量限制） |
| 空状态 | 显示"暂无记录"图标提示 |

> 选择记录确定后触发服务端冲突校验 → 详见 `domain/add-to-space-rules.md`

### 本地文件上传规则

| 类型 | 配置弹窗 | 上传后位置 | 消耗转写时长 |
|------|----------|-----------|-------------|
| 音频 | ✅ 有（区分说话人，**无**添加到空间选项） | Record 列表添加原文件，此 Space 添加快捷方式 | ✅ 消耗 |
| 文档 | ❌ 无弹窗，直接加入上传队列 | 仅此 Space（作为原文件） | ❌ 不消耗 |

**音频说话人开关**与录音设置同步，但修改不影响录音设置；关闭弹窗再次打开恢复为录音设置状态。

### 文件卡片显示规则

> 卡片通用显示规则（主标题/副标题/图标/时间/时长/红点等）→ 详见 `domain/record-common-rules.md` 第三~七节。以下为 Space 内的专属差异：

| Space 专属差异 | 说明 |
|---------------|------|
| 快捷方式图标 | 音频快捷方式卡片标题左侧额外展示**快捷方式图标**标识 |
| 文档文件大小 | 文档卡片额外展示文件大小（B / KB / MB，不显示小数点） |
| 文档卡片 | 文档文件仅在 Space 内展示，记录列表无此卡片类型 |

## 四、快捷方式置灰规则

| 触发场景 | 置灰表现 | 操作限制 |
|----------|---------|---------|
| 原文件被删除到回收站 | Space 内对应快捷方式置灰 | 点击提示「原文件已被删除」，不可进入详情 |
| 置灰快捷方式左滑 | 仅显示「移除」按钮 | 点击移除无二次确认，直接删除，不进回收站 |
| 原文件从回收站恢复 | 置灰快捷方式恢复正常 | 可正常点击进入详情 |

## 五、Space 内排序

> 4 种排序方式（创建时间/修改时间 × 升序/降序）与 Record 列表通用，详见 `domain/record-common-rules.md` 第五节。默认创建时间降序。Record 和 Space 各自独立记忆排序状态。

## 六、Space 内左滑快捷操作

| 文件类型 | 操作 | 行为 |
|----------|------|------|
| 音频快捷方式 | 移除快捷方式 | 移除到回收站，原文件不受影响 |
| 文档原文件 | 删除原文件 | 删除到回收站 |

## 七、Space 内批量操作

### 进入与退出

| 操作 | 说明 |
|------|------|
| 进入 | 长按任意记录卡片 |
| 退出 | 点击退出按钮 / Android 系统返回手势 / 关闭冲突弹窗后自动退出 |
| 解析中/解析失败记录 | **置灰不可选择，但保留卡片 UI**（进入/退出/置灰逻辑与 Record 批量一致） |
| 底部按钮 | 「导出」+「删除」两个按钮（Record 多一个「添加到」） |
| 未选择 | 底部按钮置灰不可点击 |

### 导出类型（5 种，比 Record 多"导出源文件"）

| 操作 | 说明 |
|------|------|
| 导出音频 | 快捷方式可导出；文档记录无音频时跳过 |
| 导出源文件 | **仅文档文件**可导出（pdf/docx/txt/markdown）；快捷方式不可导出源文件 |
| 导出转写 | txt / docx / pdf（含时间戳/说话人选项）；文档文件不可导出转写 |
| 导出摘要 | txt / markdown / docx / pdf |
| 导出思维导图 | markdown / pdf（**无 jpeg**）|

### 批量删除

| 文件类型 | 弹窗类型 | 行为 |
|----------|----------|------|
| 全部为文档原文件 | 「确认删除选中的来源吗？」 | 删除到回收站 |
| 全部为快捷方式 | 「确认移除选中的来源吗？」+ 勾选项「同时删除原文件」 | 默认勾选删除原文件 |
| 混合类型 | 「确认删除选中的来源吗？」+ 勾选项「同时删除原文件」 | 默认勾选删除原文件 |

## 八、测试关注点

- Space 数量套餐上限（Basic:10 / Pro:20 / Ultra:40）
- 单 Space 文件套餐上限（Basic:10 / Pro/Ultra:50）
- 未上云记录不可添加到 Space
- 添加到空间弹窗/冲突处理/回收站恢复优先 → 详见 `domain/add-to-space-rules.md` 测试关注点
- 快捷方式置灰：原文件删除后置灰，恢复后恢复正常；置灰左滑移除不进回收站
- 文档从 Space 删除 vs 音频移除快捷方式的行为差异
- 删除 Space 时内部记录全部进回收站
- **记录库弹窗**：未上云/解析中/解析失败记录隐藏；仅录音（无转写）记录**置灰不可选**
- Space 内上传音频 → Record 创建原文件 + Space 创建快捷方式
- Space 内上传文档 → 仅 Space 内展示，不消耗转写时长
- Space 批量导出 5 种类型（比 Record 多"导出源文件"），文档不可导出转写
- 批量模式下解析中/解析失败记录**置灰可见**（Record 和 Space 行为一致）
- **空间容量占用**：上传中/等待上传计入已用数量；解析失败/置灰快捷方式占用容量；跨设备不同步
