# 消息中心功能规格（viaim）

> 版本：V2.0.0 | 最后更新：2026-03  
> ⚠️ 注：消息类型详细内容后续持续补充完善

---

## 一、功能概述

消息中心用于展示系统/运营推送的消息，支持按地区定向推送。

---

## 二、消息推送规则（地区定向）

| 规则项 | 说明 |
|--------|------|
| 推送方式 | 服务端根据 App 当前 IP 所在地区推送对应地区消息 |
| 地区设置 | 创建消息时选择目标地区（如：北美、日本、东南亚、中东、其他；支持多选）|
| 推送逻辑 | App 实时 IP → 服务端判断所属地区 → 仅推送匹配该地区的消息 |
| 跨地区 | 用户切换网络/地区时，展示的消息随当前 IP 地区变化 |

**地区选项（创建消息时可选，支持多选）：**
- 北美
- 日本
- 东南亚
- 中东
- 其他

---

## 三、消息有效期

| 规则项 | 说明 |
|--------|------|
| 有效期设置 | 创建消息时设置有效期 |
| 过期行为 | 过期后消息不再展示（或从列表消失，具体行为待确认）|

---

## 四、消息列表操作

| 操作 | 说明 |
|------|------|
| 查看消息 | 点击消息可查看详情，未读标识消失 |
| 清除全部未读 | 列表中全部消息标志为已读 |
| 未读角标 | ✅ 有，消息中心入口展示未读数角标 |

## 六、测试关注点

- 地区推送规则：切换不同 IP（VPN 模拟不同地区），验证展示的消息是否与 IP 地区匹配
- 多地区消息：创建时选中东+其他，中东地区用户和其他地区用户均应收到
- 有效期：过期消息是否不再展示
- 未读角标：新消息到达时角标数字是否正确
- 点击消息后未读状态是否消失
- 清除全部未读：是否全部变为已读状态
- 节点验证：日本节点 / 新加坡节点
