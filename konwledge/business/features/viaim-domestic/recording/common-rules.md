# 录音通用详细规则（国内版 viaim App）

> 以下规则适用于**通话录音、音视频录音、现场录音**（三种录音共享）。
> 实时翻译与录音的差异在相关章节中标注。
> 功能支持矩阵和通用功能概览见 [README.md](README.md)。

---

## 一、区分说话人

> 通话/音视频/现场录音支持；实时翻译与录音**不支持**。

**展示规则：**
- 开启后每段转写文字前均展示**说话人名称 + 时间戳**（同一说话人连续发言形成多段，每段均展示说话人名称）
- 未开启区分说话人：每段展示**时间戳**，不展示说话人
- 说话人颜色标识按顺序排列，与视觉一致

> 通话录音双路（Pro3/Air2/Pro2）的说话人默认展示规则（我方/对方）见 [call.md](call.md)。

**切换操作：**
- 录音中点击开启：toast「正在切换多说话人识别模式」，之后内容开始区分说话人
- 录音中点击关闭：toast「已关闭多说话人识别」，之后内容只显示时间戳
- 暂停期间可开启/关闭：点击继续后新内容按当前状态生效
- 实时翻译开启时：翻译处不显示说话人，转写处说话人正常显示
- **悬浮窗**内：**不显示说话人**（无论是否开启）
- **画中画**内：**显示说话人**（与 App 主界面一致；App 关闭说话人则画中画也不显示）

**说话人名称修改：**
- 点击说话人名称 → 弹出修改弹窗 → 修改确认后，页面内**所有同名说话人全部同步修改**
- 名称输入框长度限制：**10 个字符**
- 点击取消：不修改，保持原名
- 修改后结束录音，详情页同步显示修改后的名称

**说话人字段初始化：**
- 设置页已开启区分说话人：进入录音页默认为开启状态
- 设置页未开启：进入录音页默认未开启

---

## 二、标记功能

> 通话/音视频/现场录音支持；实时翻译与录音**不支持**。

- 点击标记：手机**振动**，当前位置前 10s 内容重点显示（不足 10s 则全部标红）
- 标记持续：点击时起之后 10s 内容持续标红，10s 后停止
- **两次标记合并**：10s 内两次独立标记合并为一段连续标红
- **翻译不标记**：开启实时翻译时，只有原文被标红，译文**不**标红
- 悬浮窗内标记：规则与主界面一致；仅录音模式下标记按钮**置灰不可点击**

---

## 三、暂停功能

> 四种录音均支持暂停。

- 暂停时：计时器停止，说话不上屏，麦克风关闭
- 暂停期间可操作：切换转写语言、开启/关闭区分说话人、开启/关闭实时翻译、标记、修改标题、收起/展开功能栏、结束录音等
- 继续录音后：按暂停期间最新设置生效（语言、说话人、翻译状态）
- 暂停状态下发生异常结束（杀 App、断开连接、耳机入盒关盖）：再次开启录音为**新录音**状态

> 实时翻译与录音暂停时的播报行为见 [translate.md](translate.md)。

---

## 四、录音时长上限

> 四种录音均适用。

- 单次录音时长上限：**240 分钟**
- 距上限 **10 分钟**时上方提示「录音上限为240分钟，10分钟后将停止录音」，倒计时实时更新
- 达到上限：录音自动停止，提示「录音已达最大时长」，自动保存并跳转至详情页

---

## 五、录音中断提示

> 适用**通话/音视频/现场录音**；实时翻译与录音使用手机麦克风，无以下提示。

| 情况 | 提示内容 | 录音状态 |
|------|---------|---------|
| 网络不佳 | 「当前网络不佳，可能影响转文字准确率」| 录音继续 |
| 录音受干扰 | 「录音收到干扰，请尝试切换手机流量或连接5G WiFi」| 录音继续 |

> 蓝牙干扰中断（录音自动结束）**仅通话录音**，见 [call.md](call.md)。

---

## 六、耳机双连录音规则

> 耳机同时连接 A、B 两台设备时（**通话录音和现场录音**适用）：

| 场景 | 行为 |
|------|------|
| 通话在**本机**，开启通话录音 | 弹出「断开另一台设备连接」弹窗 |
| 开启**现场录音** | 弹出「断开另一台设备连接」弹窗 |
| 弹窗 → 点击以后再说 / X / 弹窗外 | 弹窗关闭，继续录音，**不**断开另一台设备 |
| 弹窗 → 点击「确认并记住我的选择」| 断开另一台设备；录音结束后**自动回连** |
| 已勾选「记住选择」后再次开启 | **自动断开**另一台设备，不再弹窗 |
| 通话**不在本机**（在另一台），开启通话录音 | 直接进入通话录音，**不弹断开弹窗** |

---

## 七、异常结束与续录规则

> 适用通话/音视频/现场录音。

| 异常情况 | 即时行为 | 20s 内回连 | 20s 后回连 |
|--------|---------|----------|----------|
| 关闭蓝牙 | 提示录音异常结束，自动保存已录部分 | 可续录（继续录音状态）| 直接回到详情页 |
| 耳机入盒关盖 | 提示录音异常结束，自动保存已录部分 | 可续录（继续录音状态）| 直接回到详情页 |

---

## 八、记录保存规则

### 通话/音视频/现场录音

| 录音模式 | 结束方式 | 时长 < 10s | 时长 ≥ 10s |
|--------|---------|----------|----------|
| 实时录音（无转写内容）| 手动结束 | **不保存** | 保存 |
| 实时录音（无转写内容）| 杀掉 App | **不保存** | 保存 |
| 实时录音（无转写内容）| 入盒关盖/关蓝牙 | **不保存** | 保存 |
| 仅录音 | 手动结束 | 保存（无论多短）| 保存 |
| 仅录音 | 杀掉 App | **保存** | 保存 |
| 实时录音（有转写内容）| 任意方式 | 保存 | 保存 |

### 实时翻译与录音（无仅录音模式）

| 内容状态 | 结束方式 | 时长 < 10s | 时长 ≥ 10s |
|--------|---------|----------|----------|
| 无转写内容 | 手动结束 | **不保存** | 保存 |
| 无转写内容 | 杀掉 App | **不保存** | **不保存** |
| 无转写内容 | 入盒关盖/关蓝牙 | **不保存** | 保存 |
| 有转写内容 | 任意方式 | 保存 | 保存 |

> 关键差异：实时翻译与录音杀进程时，**无转写内容则无论时长均不保存**；有转写内容则保存。

---

## 九、仅录音模式

> 通话/音视频/现场录音支持仅录音模式；实时翻译与录音**无此模式**。

| 功能/行为 | 说明 |
|----------|------|
| 进入条件 | 设置页未开启实时录音转文字 |
| UI 结构 | 上方计时器 + 标题 → 中部波形条 + 「开启实时转写」按钮 → 下方功能栏 |
| 开启实时转写入口 | ① 页面中间「开启实时转写」按钮 ② 功能栏「实时转写」按钮 → 均弹出语言选择弹窗 |
| 标记按钮 | **置灰不可点击** |
| iOS 画中画 | **不支持** |
| Android 悬浮窗 | 支持；悬浮窗内可选择语言开启实时转写 |

## 十、平台专属功能

### Android 悬浮窗

| 功能 | 说明 |
|------|------|
| 开启后 App 可小窗显示 | 需开启悬浮窗权限 |
| 转文字语种选择 | 悬浮窗内可选择转写语种 |
| 调整字号 | 点击「a」调小，点击「A」调大（仅对悬浮窗生效）|
| 标记 | 点击标记图标，前后 10s 内容标红 |
| 悬浮窗尺寸 | 可调整悬浮窗大小 |
| 结束录音 | 点击结束图标结束录音 |
| 静音（通话录音）| 静音后对方听不到我方说话，录音也不录我方音频 |
| 说话人展示 | 悬浮窗内**不展示说话人** |

### iOS 画中画

| 录音类型 | 实时转写状态 | 画中画支持 |
|--------|-----------|---------|
| 通话录音 | 已开启转写 | ✅ 支持，小窗查看转写翻译内容，可恢复全屏 |
| 通话录音 | 仅录音模式 | ❌ **不支持** |
| 音视频录音 | 已开启转写 | ✅ 支持 |
| 音视频录音 | 仅录音模式 | ❌ **不支持** |
| 现场录音 | 已开启转写 | ✅ 支持 |
| 现场录音 | 仅录音模式 | ❌ **不支持** |
| 实时翻译与录音 | — | ✅ 支持 |

> 仅录音模式下均**不支持**画中画。
> 画中画内**展示说话人**（与 App 主界面一致）。

## 十一、音视频录音补充

| 功能 | 说明 |
|------|------|
| **被动暂停（音源停止）** | 当前播放的音视频音源停止 → 录音自动暂停；点击「继续录音」弹 toast 提示需播放音源 |
| **获取源音频标题** | 音视频录音自动获取当前播放的音频/视频标题作为录音标题；此标题等同手动修改，结束后**不生成智能标题** |

---

## 十二、现场录音说明

> 现场录音**无独有功能**，所有规则与通用规则完全一致。
> 耳机双连时触发断开弹窗（见第六章）。
