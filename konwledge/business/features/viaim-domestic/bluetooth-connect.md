# 耳机连接流程规格（国内版 viaim App）

> 版本：V1.0.0 | 最后更新：2026-04
> 耳机型号与设置功能详见 `features/viaim-domestic/bluetooth-headset.md`

---

## 一、首次连接流程

### 1.1 权限要求（按平台）

| 平台 | 所需权限/服务 |
|------|-------------|
| iOS | 蓝牙权限（Bluetooth）|
| Android 12+ | 附近的设备权限（Nearby Devices）|
| Android 12- | 位置信息权限（Location）+ 定位服务已开启 |

### 1.2 权限引导规则

**iOS：**

| 权限状态 | 触发行为 |
|---------|---------|
| 未获取蓝牙权限，点击添加设备 | 弹出系统蓝牙授权弹窗 |
| 首次点击不允许 | **不弹**「去设置」弹窗 |
| 非首次拒绝/关闭后 | 弹出「去设置」弹窗；点击「去设置」跳转系统设置 |
| 已授权 + 蓝牙已开启 | 直接进入搜索设备页面 |
| 已授权 + 蓝牙未开启 | 拉起「请打开蓝牙」页面；开启蓝牙后点击继续 |

**Android 12+：**

| 权限状态 | 触发行为 |
|---------|---------|
| 未获取附近设备权限，点击添加设备 | 弹出「需要附近的设备」弹窗 |
| 点击「去开启权限」 | 弹出系统授权选项 |
| 已授权 + 蓝牙已开启 | 进入搜索设备页面 |
| 已授权 + 蓝牙未开启 | 拉起「请打开蓝牙」页面 |

**Android 12-：**

| 权限状态 | 触发行为 |
|---------|---------|
| 未获取位置权限，点击添加设备 | 弹出「需要位置信息权限」弹窗 |
| 已授权 + 定位未开启 | 弹出「需要开启定位服务」弹窗 |
| 已授权 + 定位已开启 + 蓝牙已开启 | 进入搜索设备页面 |

### 1.3 气泡引导

| 场景 | 行为 |
|------|------|
| 首次登录 App | 首页显示气泡引导「在这里添加你的 viaim 设备」|
| 点击添加设备按钮 | 气泡引导消失 |
| 退出后再次登录 | **不再显示**（仅首次登录才显示）|
| 卸载重装后 | 重新显示气泡引导 |

### 1.4 搜索设备流程

```
点击添加设备 → 进入搜索设备中页面
  ├─ 搜索中：左上角返回 → 记录页；底部「手动添加」→ 未发现设备页
  ├─ 搜索中关闭系统蓝牙 → 搜索超时
  ├─ 长时间未搜到 → 搜索超时页
  └─ Android：搜索到设备 → 弹出连接中弹窗
```

**搜索超时页：**
- 底部显示「重新扫描」和「手动添加」按钮

**未发现设备页：**
- 点击各型号耳机 → 弹出该型号「如何连接」引导弹窗
- 弹窗内点击「前往设置」→ 系统蓝牙设置

### 1.5 连接中与失败处理

| 场景 | 行为 |
|------|------|
| 连接中切换 App 到后台再切回 | 连接状态保持，继续连接 |
| 连接中系统蓝牙断连 | 弹出「无法连接到」弹窗 |
| 弹窗内点击「重新连接」| 再次进入连接中弹窗 |
| 累计重连 3 次仍失败 | 弹窗显示「重置蓝牙配对」按钮 |

### 1.6 首次连接成功

- 首页顶部显示连接耳机动画
- 弹出「欢迎使用」弹窗（图片与实际连接型号一致）
- 欢迎弹窗按**设备维度**触发（以耳机SN为key记录，每个物理耳机首次连接均弹出）
- 点击弹窗外 → **不关闭弹窗**
- 点击「开始使用」→ 进入耳机更多设置页面

**首次连接后的引导（视耳机型号）：**
- 入耳式耳机（Pro3/Pro2/Nano+）：弹出耳塞贴合度检测弹窗
- 开放式耳机（Air2/VIAIM Air）：不弹贴合度检测
- **非首次连接**（本地已有该耳机弹窗记录）：不弹欢迎弹窗，仅 Toast 提示

## 二、非首次连接与自动重连

### 2.1 App 打开时自动重连

- 前提：曾经连接过耳机 + 所有权限已授权 + 系统蓝牙已开启
- 首页左上角显示「正在连接」

### 2.2 重连结果

| 结果 | 首页表现 |
|------|---------|
| 搜索到设备 + 连接成功 | 显示耳机连接成功动画 |
| 搜索到设备 + 连接失败 | 状态变为「未连接」|
| 搜索超时（未搜到设备）| 状态变为「未连接」|

> iOS：蓝牙无连接设备时进入 App 直接显示「未连接」，不触发搜索。

### 2.3 首页连接状态交互

| 点击状态 | 行为 |
|---------|------|
| 点击「正在搜索/连接中」| 进入耳机设置缺省页，继续搜索 |
| 点击「未连接」| 进入耳机设置缺省页，触发自动重连 |

### 2.4 耳机设置页中的重连

```
进入耳机设置页 → 自动触发重连搜索
  ├─ 搜索到设备 + 连接成功 → 页面更新显示设置项
  ├─ 搜索到设备 + 连接失败 → 弹出「无法连接到[设备名]」弹窗
  ├─ 重连搜索超时 → 弹出上次连接耳机的「如何连接」引导弹窗
  └─ 重连失败 → 连接状态区域显示「重新连接」按钮
```

### 2.5 耳机设置页添加设备按钮

| 当前状态 | 点击添加设备 |
|---------|------------|
| 未连接 | 进入搜索设备页面 |
| 已连接 | 弹「请先断开」弹窗 |
| 搜索中 | 按钮置灰，点击无响应 |

## 三、MFI 连接流程（Air2/opennote 专属）

> Air2（opennote）使用 MFI（Made for iPhone）协议，首次连接时走专属的配对发现页面。

| 耳机状态 | 行为 |
|---------|------|
| 未开盖 | 搜索超时，无法搜索到 |
| 开盖但**未进入配对状态** | 搜索到 opennote，进入「发现 opennote 设备」页面；点击配对 → 配对列表**为空** |
| 开盖 + **长按进入配对状态** | 配对列表显示耳机名称，可选择连接 |
| 多个 opennote 同时配对 | 配对列表显示所有耳机名称 |

- MFI 发现页有 **15 秒超时限制**
- 通过系统蓝牙直连 opennote → 弹出「viaim 请求通信」弹窗，点击「允许」→ 拉起 App

## 四、测试关注点

**首次连接：**
- iOS 首次拒绝蓝牙权限：**不弹**「去设置」弹窗
- Android 12- 需位置权限 + 定位服务双重检查
- 欢迎使用弹窗：**按设备维度**，点击弹窗外不关闭
- 重连 3 次失败才显示「重置蓝牙配对」按钮

**非首次连接与复连：**
- iOS 蓝牙无连接设备时进入 App 直接显示「未连接」，不触发搜索
- 耳机设置页搜索超时 → 弹上次连接耳机引导弹窗
- 断开时设备详情页设置项**即时**变为缺省状态

**MFI（Air2/opennote）：**
- opennote 必须开盖 + 长按进入配对状态，配对列表才有耳机名称
- 通过系统蓝牙直连 opennote：点击「取消」不会拉起 App
