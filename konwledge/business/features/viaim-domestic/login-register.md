# 登录注册功能规格（国内版 viaim App）

> 版本：V1.0.0 | 最后更新：2026-04

---

## 一、闪屏页与用户协议

| 场景 | 行为 |
|------|------|
| 首次安装打开 App | 弹出「重要提醒」弹窗，包含隐私政策和用户服务协议链接 |
| 点击「同意并继续」 | 进入登录注册页 |
| 点击「不同意」 | 直接退出 App |
| 点击隐私政策/服务协议 | 跳转对应页面查看内容（中文版） |

> 国内版隐私政策遵循《个人信息保护法》（PIPL），文案为中文版。

---

## 二、登录注册首页

| 元素 | 说明 |
|------|------|
| 登录/注册入口 | 手机号登录（首次使用自动注册） |
| 底部用户协议勾选 | **未勾选时点击登录按钮无法进入下一步**，条款文字抖动提示 |

> 国内版不支持邮箱登录、Google 登录、Apple 登录。

---

## 三、手机号登录/注册流程

```
手机号输入页
  → 发送短信验证码
  → 验证码输入页（60s 倒计时）
  → 登录成功 / 首次登录自动完成注册
```

### 3.1 手机号输入规则

| 规则项 | 说明 |
|--------|------|
| 键盘类型 | 数字键盘 |
| 区号选择 | 支持选择国际区号（默认 +86） |
| 最大长度 | 11 个字符，超出不显示 |
| 格式校验 | 格式不正确则提示「请输入正确的手机号码」|
| 返回上一页 | 输入框清空 |

### 3.2 短信验证码规则

| 规则项 | 说明 |
|--------|------|
| 发送 | 输入合法手机号后点击「获取验证码」|
| 倒计时 | 60s，倒计时内「重新发送」置灰 |
| 验证码有效期 | **5 分钟**，过期后需重新获取 |
| 验证码错误 | 输入框抖动变红，清空输入框，Toast 提示「验证码错误，请重试」|
| 重新发送 | 倒计时结束后可点击重新发送，重新开始 60s 倒计时 |
| 返回上一页 | 验证码输入框清空 |
| 再次进入验证码页 | 倒计时重新开始（60s 内返回再进入也会重发）|

## 四、扫码登录（App 扫描桌面端二维码）

| 场景 | 行为 |
|------|------|
| 桌面端显示登录二维码 | 使用已登录的 App 扫描桌面端二维码 |
| 扫码成功 | App 弹出确认登录弹窗（含桌面端设备信息）|
| 点击「确认登录」 | 桌面端完成登录 |
| 点击「取消」 | 取消此次扫码登录，桌面端二维码恢复等待状态 |
| 二维码过期 | 提示「二维码已过期，请刷新」|
| 未登录状态扫码 | 提示需先登录 App |

---

## 五、首次登录引导

### 新注册用户

| 规则 | 说明 |
|------|------|
| 录音记录列表 | 自动创建示例文件，供新用户了解功能 |
| 支持编辑和删除 | 示例文件支持编辑和删除操作 |
| 删除后不重建 | 删除后再次登录不会重新创建 |

### 登录同步行为

| 场景 | 行为 |
|------|------|
| 退出重新登录 | 显示"正在同步记录"页面，同步成功后展示记录 |
| 登录状态下升级安装 App | 不显示同步页面，直接展示记录列表 |

---

## 六、退出登录 / 多端互踢

| 场景 | 行为 |
|------|------|
| 主动退出登录 | 清除本地登录态，返回登录页 |
| 账号在其他设备登录 | 当前设备弹出「账号已在其他设备登录」提示，返回登录页（多端互踢）|

---

## 七、测试关注点

- 首次安装弹出重要提醒弹窗，「不同意」直接退出 App
- 未勾选协议点击登录：条款抖动，不跳转
- 手机号输入格式校验：格式错误时提示，不发送验证码
- 验证码 60s 倒计时；5 分钟过期；验证错误时清空输入框并抖动
- 返回验证码页倒计时重置（即使 60s 内返回）
- 首次登录：自动注册并创建示例文件，删除后不重建
- 退出重登：显示同步页；升级安装不显示同步
- 扫码登录：需确认弹窗，取消不触发桌面端登录
- 多端互踢：当前设备被踢后返回登录页
