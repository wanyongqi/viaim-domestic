# 套餐权益（viaim）

> 版本：V2.0.2 | 最后更新：2026-04

## 零、未激活会员体系（V2.0.2 新增）

### 会员状态层级

```
未激活会员 → Basic → Pro → Ultra
```

V2.0.2 起新注册用户默认为「未激活会员」，需通过绑定设备或购买权益激活。

### 未激活会员定义

| 条件 | 是否为未激活会员 |
|------|---------------|
| V2.0.2 上线后新注册账号（不论注册端/App版本）| ✅ |
| V2.0.2 前已注册的老用户（已为 Basic）| ❌ 自动激活，不弹解锁弹窗 |

### 老版本用户升级兼容规则

V2.0.2 前没有绑定流程，老版本用户连接耳机后即为 Basic 会员可正常上传。

**服务端修正后的规则：** 老版本App注册的账号，曾连接过耳机 → 升级到V2.0.2后自动识别为**Basic会员**，无需绑定设备。

> **历史缺陷背景**：服务端上线时初始逻辑要求必须完成设备绑定才激活 Basic，但老版本App无绑定流程，导致老用户升级后被误判为未激活会员无法上传。修正后改为：老版本连接过耳机的用户直接视为已激活 Basic。

### 未激活会员功能限制

| 功能 | 是否可用 |
|------|---------|
| 查看已有数据 | ✅ |
| Vitana 对话 | ✅ |
| Record 添加记录 | ❌ 弹出解锁 Basic 弹窗 |
| Space 添加来源 | ❌ 弹出解锁 Basic 弹窗 |
| 上传音频/文档 | ❌ 不可上传 |

### 解锁 Basic 弹窗

**触发入口（3个）：**
- Record 页添加记录
- Space 页添加来源
- 我的页面点击「解锁 Basic」

**弹窗操作：**

| 操作 | 行为 |
|------|------|
| 点击添加设备 | 跳转耳机搜索连接流程 → 绑定成功 → 激活 Basic |
| 点击直接购买权益 | 弹出购买权益弹窗（默认展示 Pro Tab）|
| X / 顶部下拉 / 弹窗外关闭 | 回到触发前页面 |

### 激活路径

| 激活方式 | 结果 |
|---------|------|
| 绑定设备成功 | 激活 Basic，可正常上传 |
| 点击添加设备后退出绑定 | 激活失败，仍为未激活 |
| 购买 Pro/Ultra | 直接激活对应套餐 |

### 降级回退规则

| 场景 | 降级后状态 | 原因 |
|------|----------|------|
| 已激活 Basic → 升级 Pro/Ultra → 到期 | **Basic**（非未激活）| 曾通过绑定设备激活过 Basic |
| 未激活 → 直接购买 Pro/Ultra → 到期 | **未激活会员** | 未曾绑定设备，无 Basic 激活记录 |
| Basic 会员 → 解绑所有设备 | **仍为 Basic** | 一旦激活不回退 |

### 我的页面展示

| 会员状态 | 我的页面展示 |
|---------|------------|
| 未激活会员 | 解锁 Basic 蓝色按钮 + 查看其他权益橙色文字 |
| 已激活会员（Basic/Pro/Ultra）| 会员卡片，无解锁入口 |

**操作：**
- 点击「查看其他权益」→ 跳转订阅与方案页面
- 点击「解锁 Basic」→ 弹出解锁 Basic 弹窗

## 三、订阅管理规则

**升级路径：** Basic → Pro ✅ / Basic → Ultra ✅ / Pro → Ultra ✅

**套餐变更限制（不可降级）：**

| 当前套餐 | 不可操作 |
|--------|---------|
| Ultra 月 / Ultra 年 | 不可选 Pro 月/Pro 年/Basic |
| Pro 月 | 不可选 Pro 年/Basic |
| Pro 年 | 不可选 Pro 月/Basic |

**支付渠道：**
- iOS：Apple Store
- Android：Google Play
- 桌面端 / Web 端：**空中云汇（Airwallex）**

**自动续订触发机制（App 端）：**
- 触发条件：当前有效期**剩余时间 ≤ 2小时**且达到整点时，系统扫描触发续订扣款
- 扣款成功：有效期延长（月套餐 +31天，年套餐 +366天），月套餐用量同步重置为0
- 扣款失败 / 取消订阅到期：套餐降级为 Basic，用量重置为0/600

**升级有效期折算公式（Pro → Ultra）：**

> N = 当前套餐剩余天数（**不含当天**）

| 升级路径 | 有效期公式 |
|--------|----------|
| Pro 年 → Ultra 年 | 当前日期 + 366天 + (Pro年费/366×N)÷(Ultra年费/366)，结果取整 |
| Pro 年 → Ultra 月 | 当前日期 + 31天 + (Pro年费/366×N)÷(Ultra月费/31)，结果取整 |
| Pro 月 → Ultra 年 | 当前日期 + 366天 + (Pro月费/31×N)÷(Ultra年费/366)，结果取整 |
| Pro 月 → Ultra 月 | 当前日期 + 31天 + (Pro月费/31×N)÷(Ultra月费/31)，结果取整 |

**多端同步与取消恢复：**
- 任意端（iOS/Android/桌面端）的订阅变更（订阅/升级/取消）**同步至所有其他端**
- 已取消但未过期时可点击「恢复订阅」，恢复后状态变回「自动续费」，多端同步

**退款与异常：**
- Apple / Google Play 退款成功 → 套餐降级为 Basic
- 支付失败 / 用户取消支付 → 套餐保持不变
- 支付中网络断开 → 提示网络异常；重新进入后根据服务端订单状态显示实际套餐
- 支付成功但套餐未生效 → 账单管理中显示支付记录，需联系客服处理

## 四、转写时长扣除与返还机制（仅 Basic/Pro）

> Ultra 用户无转写时长限制，不涉及扣除逻辑。

### 4.1 扣除时机与扣除方

| 状态 | 扣除方 | 多端同步 | 说明 |
|------|--------|---------|------|
| 等待上传 | **客户端** | ❌ 不同步 | 点击确认进入上传队列时客户端本地扣除 |
| 上传中 | **服务端** | ✅ 同步 | 等待上传 → 上传中时，服务端接管扣除，客户端释放（不重复扣除） |
| 解析中 | **服务端** | ✅ 同步 | 服务端持续占用 |
| 实时录音转写 | **服务端** | ✅ 同步 | 录音过程中实时扣除，时长随录音进度减少 |

### 4.2 各入口剩余时长展示差异

| 入口 | 扣除范围 | 原因 |
|------|---------|------|
| 添加记录弹窗 / 添加来源弹窗 | 等待上传 + 上传中 + 解析中 | 弹窗内展示客户端本地计算值 |
| 设置页面 / 转文字弹窗 / 实时转写界面 | 上传中 + 解析中（**不含**等待上传） | 仅展示服务端同步的剩余时长 |

### 4.3 时长返还规则

| 场景 | 返还行为 |
|------|---------|
| 等待上传 → 点击取消 | 客户端返还 |
| 等待上传 → 上传失败 | 客户端返还 |
| 上传中 → 点击取消 | 服务端返还（无 Toast） |
| 上传中 → 上传失败 | 服务端返还 |
| 解析成功 | **不返还**，正常扣除 |
| 解析失败 | 服务端返还（无 Toast） |
| 转写无文本（无语音内容） | **不算解析失败，不返还** |
| 仅录音 → 转文字成功 | 不返还 |
| 仅录音 → 转文字失败 | 返还预扣除时长 |
| 重新转写成功 | 不返还 |
| 重新转写失败 | 返还预扣除时长 |

### 4.4 上传前校验规则

| 条件 | 结果 |
|------|------|
| 消耗时长 ≤ 剩余时长 | 可上传，进入队列 |
| 消耗时长 > 剩余时长 | 弹出升级权益弹窗，不可上传 |
| 消耗时长 = 剩余时长（边界） | 可上传 |

**精度**：展示为分钟，向上取整。

**Ultra 用户**不显示"将消耗/剩余"文案。

### 4.5 App 重启与异常场景

| 场景 | 行为 |
|------|------|
| 等待上传 → 杀进程重启 | 变为上传失败，客户端返还 |
| 上传中 → 杀进程重启 | 变为上传失败，启动 App 上报服务端，服务端立即返还 |
| 解析中 → 杀进程 | 服务端继续解析；成功则正常扣除，失败则返还 |
| 杀进程后一直不重启（超 2 小时）| 服务端每小时检测，超 2 小时未完成的标记失败并返还 |
| 上传中套餐降级导致时长不足 | 已在上传中/解析中的记录继续处理不受影响 |
| 套餐过期，上传中的文件 | 继续完成上传和解析 |
| 套餐过期，等待上传的文件 | 变为上传中时服务端校验失败 → 上传失败（无提示） |

### 4.6 非实时转写扣除

详情页对仅录音/转写无文本/已有转写记录发起转写时：
- 弹出消耗时长确认弹窗
- 点击确认后**预扣除**时长
- 成功 → 正常扣除；失败 → 返还

### 4.7 闪录下载 + 自动转写 + 时长不足

| 场景 | 行为 |
|------|------|
| 开启自动转写 + 时长充足 | 上云成功后自动转写 |
| 开启自动转写 + 时长不足 | 上云成功后**不转写**，显示仅录音状态，无弹窗/Toast |
| 未开启自动转写 + 时长不足 | 正常仅录音，无提示 |

## 八、测试关注点

**未激活会员（V2.0.2）：**
- 新注册用户默认为未激活会员，老用户升级V2.0.2自动激活Basic（不弹解锁弹窗）
- 解锁弹窗三个触发入口（Record添加记录/Space添加来源/我的页面）行为一致
- 激活路径：绑定设备 vs 购买权益，中途退出绑定不激活
- 降级回退区分「曾绑定设备」和「未曾绑定设备」，前者回到Basic，后者回到未激活
- Basic会员解绑所有设备后仍为Basic（一旦激活不回退）
- 未激活会员可查看已有数据和Vitana对话，但不可添加记录/来源/上传

- Basic 超限弹窗 vs Pro/Ultra 静默阻断（注意 Space文件数/音频上传/Ultra模板 的特殊行为）
- 祖父条款：超限数量保留，不可新建，删除后才释放
- 套餐升级后权益立即生效；**已用转写时长不重置**（续订后才重置）
- 升级弹窗：Basic 触发显示 Pro+Ultra Tab；Pro 超限仅显示 Ultra Tab；默认月付
- 自动续订：剩余 ≤ 2小时 + 整点触发（非到期瞬间）
- Pro 升 Ultra 有效期折算：按剩余价值换算（有具体公式，需验证精度）
- 跨渠道互斥：Apple/Google 订阅不可重复；不同账号切换不可购买
- 支付渠道：iOS → Apple Store；Android → Google Play；桌面/Web → 空中云汇
- **转写时长扣除**：等待上传(客户端) vs 上传中/解析中(服务端)，扣除方和多端同步逻辑不同
- **各入口剩余时长展示**：添加记录/来源弹窗含等待上传；其他入口不含（仅服务端值）
- **时长返还**：等待上传取消/失败返还；解析成功不返还；转写无文本不算失败不返还
- **App 重启**：等待上传变失败(客户端返还)；上传中变失败(服务端返还)；解析中继续处理
- **服务端兜底**：杀进程 2 小时未完成 → 服务端标记失败并返还
