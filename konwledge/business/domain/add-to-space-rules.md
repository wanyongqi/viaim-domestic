# 添加到空间规则（viaim）

> 版本：V2.0.0 | 最后更新：2026-03
> 适用场景：Record 列表左滑/批量添加、Record 上传时选择空间、Space 内添加来源、回收站恢复（含原 Space 已删除时弹 Space 选择弹窗）
> 校验方式：服务端校验返回结果，客户端展示对应弹窗

---

## 一、触发入口

| 触发入口 | 操作流程 |
|---------|---------|
| Record 列表左滑 → 添加到 | 弹出添加到空间弹窗 → 选择目标空间 → 确定 → 服务端校验 |
| Record 批量 → 添加到 | 弹出添加到空间弹窗 → 选择目标空间（可多选）→ 确定 → 服务端校验 |
| Record 上传配置 → 添加到空间 | 弹出添加到空间弹窗 → 选择目标空间 → 上传成功后服务端校验 |
| Space 内添加来源 → 记录库 | 展示记录库列表（规则见 `features/viaim/space.md`）→ 选择记录 → 确定 → 服务端校验 |
| 回收站恢复（原 Space 已删除）| 弹出添加到空间弹窗 → 选择新的目标空间 → 确定 → 服务端校验 |

> 所有入口最终都通过**添加到空间弹窗**完成操作，弹窗展示空间列表，用户选择空间后提交服务端校验。

---

## 二、添加到空间弹窗

| 内容 | 说明 |
|------|------|
| 空间列表 | 展示图标 / 名称 / 记录数量和容量 / 创建时间，按创建时间倒序 |
| 容量已满标识 | Basic 空间已满 10 个记录 / Pro-Ultra 已满 50 个记录 → 置灰不可选，容量标红（如 10/10）|
| 新建空间入口 | 弹窗内可直接新建空间，新建成功后置顶且自动选中 |
| 空列表 | 无空间时显示空状态 |
| 多选 | 可同时勾选多个空间 |

### 套餐上限与新建

| 套餐 | 新建 Space 上限 | 触发行为 |
|------|---------------|---------|
| Basic | 10 个 | 第 10 个满时点击新建 → 弹出上限弹窗；Basic 弹窗含「升级权益」按钮 |
| Pro | 20 个 | 同上 |
| Ultra | 40 个 | 同上，弹窗无升级按钮（已最高套餐）|

---

## 三、回收站恢复优先规则

添加记录到空间时，系统首先检查该空间回收站中是否存在该记录的快捷方式。若存在，**优先从回收站恢复**而非创建新快捷方式。

| 场景 | 行为 |
|------|------|
| 记录 A 的快捷方式在空间 X 回收站中 | 添加到空间 X → 从回收站恢复，回收站中该快捷方式消失 |
| 记录 A 的快捷方式在空间 Y 回收站中（非目标空间）| 添加到空间 X → 在空间 X 新建快捷方式，空间 Y 回收站不受影响 |
| Record 上传新文件到回收站有重名快捷方式的空间 | 新建快捷方式，回收站中的快捷方式保留不变 |
| 批量添加：部分在回收站 + 部分重复 + 部分新增 | 回收站的先恢复 → 再判断重复 → 新增剩余记录 |

**恢复时间戳规则**：
- 创建时间：**保持原创建时间不变**
- 修改时间：**更新为恢复时的当前时间**

---

## 四、添加冲突处理规则

**容量判断原则**：去除重复记录后，判断剩余待添加数量是否超出空间剩余容量。部分已存在 + 剩余可添加 = 不弹容量弹窗（已存在跳过后不超容量即可）。

| 场景 | 行为 |
|------|------|
| 添加单个空间，记录已存在 | 弹出「记录已存在」弹窗，添加失败 |
| 批量添加单个空间，部分已存在 + 剩余可添加 | 弹「部分空间添加失败」弹窗；已存在跳过，其余成功 |
| 批量添加单个空间，部分已存在 + 剩余数量 = 容量 | 添加成功**无弹窗**；已存在跳过，其余刚好添满 |
| 批量添加单个空间，部分已存在 + 剩余超容量 | 弹「存储空间已满」弹窗，全部添加失败 |
| 批量添加单个空间，全部已存在 | 弹「记录已存在」弹窗，全部跳过 |
| 批量添加单个空间，均不存在 + 容量足够 | 全部添加成功，无弹窗 |
| 批量添加单个空间，均不存在 + 容量不足 | 弹出「存储空间已满」弹窗，全部添加失败 |
| 批量添加多个空间，部分空间容量不足 | 弹「部分空间添加失败」弹窗；容量足够的空间成功，不足的失败 |
| 批量添加多个空间，部分空间记录全部重复 | 弹「部分空间添加失败」弹窗；重复空间失败，其余成功 |
| 批量添加多个空间，全部空间容量不足 | 弹「存储空间已满」弹窗，全部失败 |
| 批量添加多个空间，全部空间全部重复 | 弹「记录已存在」弹窗 |
| 批量添加多个空间，所有空间容量足够 + 全部不重复 | 全部添加成功，无弹窗 |

**弹窗优先级**（多空间场景）：部分空间添加失败 > 存储空间已满 > 记录已存在

## 五、冲突弹窗详细规格

### 「记录已存在」弹窗

| 属性 | 说明 |
|------|------|
| 文案 | "选中的记录已存在，请勿重新上传" |
| 按钮 | 「知道了」 |
| 关闭方式 | 知道了按钮 / X 按钮 / 按住下拉 / 点击弹窗外区域 |

### 「部分空间添加失败」弹窗

| 属性 | 说明 |
|------|------|
| 文案 | "已成功添加到可用空间；部分空间容量已满，或记录重复，请清理空间后再试" |
| 按钮 | 「知道了」 |
| 关闭方式 | 知道了按钮 / X 按钮 / 按住下拉 / 点击弹窗外区域 |

### 「存储空间已满」弹窗

| 用户套餐 | 文案 | 按钮 | 关闭方式 |
|---------|------|------|---------|
| Basic | "你的空间存储已满，请清理部分文件后上传或升级权益" | 「升级权益」 | X 按钮 / 按住下拉 / 点击弹窗外区域 |
| Pro / Ultra | "你的空间存储已满，请清理部分无用文件，以便继续上传" | 「知道了」 | 知道了按钮 / X 按钮 / 按住下拉 / 点击弹窗外区域 |

**Basic 升级权益跳转行为**：点击「升级权益」→ 弹出升级弹窗，展示 Pro 升级 Tab 页面，**空间数量权益卡片置顶选中**。

### 通用交互规则

- 批量模式下执行添加操作后，关闭冲突弹窗 → **退出批量模式**，返回正常列表
- 回收站恢复后**多端同步**：本端恢复操作完成后，其他端同步看到快捷方式从回收站恢复到空间

---

## 六、测试关注点

- **添加到空间弹窗**：容量已满的空间置灰不可选、容量标红；新建空间后置顶且自动选中
- **回收站恢复优先**：若回收站有该空间的快捷方式则恢复而非新建；恢复后创建时间不变、修改时间更新
- **回收站恢复多端同步**：本端恢复后其他端同步可见
- 单空间冲突：部分已存在 + 剩余数量恰好 = 容量时**无弹窗**
- 弹窗优先级（多空间）：部分空间添加失败 > 存储空间已满 > 记录已存在
- **三种弹窗关闭方式**：知道了/X/按住下拉/弹窗外（Basic「存储空间已满」无「知道了」按钮，用升级权益替代）
- Basic 点击「升级权益」→ 弹升级弹窗展示 Pro Tab，空间数量权益卡片置顶选中
- 批量模式下关闭冲突弹窗 → 自动退出批量模式
- 回收站恢复场景：原 Space 已删除 → 弹 Space 选择弹窗
