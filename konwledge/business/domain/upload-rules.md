# 上传规则（viaim）

> 版本：V2.0.0 | 最后更新：2026-03

## 一、上传入口总览

| 入口位置 | 支持类型 | 上传配置弹窗 | 备注 |
|----------|----------|-------------|------|
| 记录列表 → 右上角导入按钮 | **仅音频** | ✅ 有（区分说话人/添加到空间/转写消耗提示） | 不支持文档 |
| Space 内 → 上传按钮 → 音频 | 音频 | ✅ 有（区分说话人/转写消耗提示，无空间选择） | 已在 Space 内 |
| Space 内 → 上传按钮 → 文档 | 文档 | ❌ 无弹窗，直接上传 | 已在 Space 内 |

**关键规则：文档文件只能从 Space 内上传，记录列表入口不支持文档上传。**

---

## 二、支持格式与限制

### 音频格式

| 格式 | 支持 |
|------|------|
| MP3 | ✅ |
| WAV | ✅ |
| M4A | ✅ |
| AAC | ✅ |
| FLAC | ✅ |
| WMA | ✅ |

**不支持格式：文件置灰，不可选择**

### 文档格式

| 格式 | 支持 |
|------|------|
| PDF | ✅ |
| DOCX | ✅ |
| TXT | ✅ |
| Markdown (.md) | ✅ |

### 大小与时长限制

| 限制项 | Basic | Pro | Ultra |
|--------|-------|-----|-------|
| 文件大小上限 | 200MB | 200MB | 200MB |
| 音频时长上限 | 1 小时 | 4 小时 | 4 小时 |
| 文档时长限制 | 无 | 无 | 无 |
| 消耗转写时长 | 音频开启转写时消耗 | 同左 | 同左 |
| 文档消耗转写时长 | ❌ 不消耗 | ❌ 不消耗 | ❌ 不消耗 |

**超限弹窗规则：**
- 音频时长和大小同时超限 → 仅显示时长超限弹窗
- 仅大小超限 → 显示大小超限弹窗

## 三、上传配置弹窗（音频）

弹出时机：选择音频文件后弹出。

| 配置项 | 说明 |
|--------|------|
| 区分说话人开关 | 开启则根据选择说话人数区分说话人 |
| 添加到空间（仅记录列表入口） | 可选择单个或多个空间，上传成功后在对应 Space 创建快捷方式 |
| 确定 | 加入上传队列开始上传 |
| X/按住下拉/弹窗外 | 关闭弹窗，取消上传 |

**上传音频无「录音实时转文字」开关，所有上传音频均自动转写。**

**文档无配置弹窗，选择后直接加入上传队列。**

### 转写时长消耗提示（添加记录 / 添加来源弹窗内）

| 套餐 | 展示内容 |
|------|---------|
| Basic / Pro | "将消耗 XX 分钟 / 剩余 XX 分钟"（无论是否开启转写均显示） |
| Ultra | **不显示**消耗/剩余文案 |

**精度**：消耗时长 = 音频时长向上取整到分钟；剩余时长 = 套餐额度 − 已使用时长（向上取整）。

**校验规则**：消耗时长 > 剩余时长 → 点击确认弹出"转写时长不足"弹窗；≤ 则正常入队。

> 详细扣除/返还机制见 `subscription.md` 第四节。

---

## 四、上传队列规则

| 规则项 | 说明 |
|--------|------|
| 同时上传数量 | 最多 3 个（**全局限制**，含 Record 和所有 Space） |
| 等待队列上限 | 最多 3 个（上传中 3 + 等待中 3 = 最多 6 个） |
| 队列满时 | Toast 提示"上传队列已满，请稍后再添加" |
| 排列顺序 | 按添加队列时间倒序排列 |

### 上传进度管理弹窗

**入口**：Record / Space 列表右上角上传按钮（有上传中/等待上传时显示上传动画；仅有失败记录时显示静态按钮；全部为空时不展示）。

**Record 和 Space 共享同一上传队列**，两个页面的上传进度管理弹窗展示相同内容。

| 卡片状态 | 展示内容 | 操作 |
|----------|----------|------|
| 上传中 | "上传中"文案 + 文件名后缀（超长省略）+ 进度条 + 百分比 + 取消按钮 | 取消 → 直接删除不进回收站，下一个开始上传 |
| 等待上传 | "等待上传"文案 + 文件名后缀（超长省略）+ 取消按钮 | 取消 → 直接删除不进回收站 |
| 上传失败 | 红色"上传失败"文案 + 文件名后缀 + 清除按钮 | 清除 → 直接删除不进回收站 |
| 上传成功 | 从进度弹窗消失，显示在记录列表/Space | — |

**上传失败记录持久化**：关闭弹窗再打开 / 切换页面 / 重启 App 依然展示。

**跨设备规则**：上传进度弹窗不同步，仅展示当前设备的上传记录。

### 文件上传与耳机上云的关系

| 规则 | 说明 |
|------|------|
| 独立队列 | 文件上传队列（最多 3 个）与耳机录音上云（最多 1 个）互不影响 |
| 同时进行 | 上传文件和耳机录音可同时进行 |

## 五、上传后状态流程

### 音频文件

上传音频**必须转写，无法关闭转写**。

| 状态流程 | 详情页内容 |
|----------|------------|
| 上传成功 → 解析中（紫色边框动画）→ 解析成功/失败 | 转写文字 + AI 生成内容（摘要/待办/脑图）|

转写处理：服务端截取音频前 60s 自动识别语种 → 按识别语种转文字。

### 文档文件

| 状态流程 | 解析处理 | 详情页内容 |
|----------|----------|------------|
| 上传成功 → 解析中（紫色边框动画）→ 解析成功/失败 | 服务端提取文档文本 | 摘要（含待办）、预览（原文）、思维导图、AI 对话 |

文档详情页无音频播放器。

### 解析失败处理

- 显示：红色边框 + 解析失败文案 + 右上角清除按钮
- 操作：点击清除直接删除（**不进回收站**）
- 限制：解析失败记录不可进入详情页 / 不可左滑 / 批量模式下**置灰不可选择，但保留卡片 UI**

## 六、展示位置规则

| 文件类型 | 记录列表 | Space 内 |
|----------|----------|----------|
| 上传音频 | ✅ 展示 | ✅ 展示（如有添加到 Space）|
| 上传文档 | ❌ 不展示 | ✅ 仅 Space 内展示 |

## 七、耳机录音上云流程

### 上云触发

| 触发方式 | 说明 |
|----------|------|
| 录音结束 | 四种录音（通话/现场/音视频/实时翻译）结束后自动进入上云流程 |
| 闪录下载完成 | 下载完成后自动进入上云流程 |

### 上云状态流程

等待上云（图标）→ 上云中（动画）→ 上云成功（录音类型图标）/ 上云失败（失败图标）

### 上云队列

| 规则 | 说明 |
|------|------|
| 并发数 | 同一时间只能 1 个记录上云 |
| 排队策略 | 先进先出，按录音生成顺序依次上云 |
| 上云失败 | UI 保留在原位置，系统自动跳过，下一条开始上云 |
| App 退出恢复 | 按列表记录展示顺序（创建时间倒序）从上到下依次上传 |

### 上云失败重试

| 触发方式 | 说明 |
|----------|------|
| 点击失败图标 | 手动触发重新上传 |
| 进入详情页 | 自动触发重新上传 |
| 切换底部 Tab | 自动触发重新上传 |
| 重试排队 | 若有其他上云中/等待上云记录 → 加到队列最尾部显示等待上云；否则立即开始 |

**网络断开/飞行模式**：上云失败，显示失败图标；网络恢复后**不会自动重试**。

### 网络切换行为

| 场景 | 行为 |
|------|------|
| 耳机录音上云：WiFi → 4G/5G | 继续上云不中断 |
| 耳机录音上云：4G/5G → WiFi | 继续上云不中断 |
| 文件上传：WiFi → 4G | **从头重传** |
| 上云/上传完全断网 | 失败，恢复网络后不自动重试 |

### 其他

- 不支持断点续传（中断/App 重启后从头上传）
- App 切后台：上云任务继续（系统允许情况下）；被系统回收后重新打开自动恢复队列
- 上云成功后自动开始生成 AI 内容

## 八、空间容量占用规则（上传场景）

### 上传中/等待上传计入空间已用数量

| 规则 | 说明 |
|------|------|
| Record 上传添加到空间 A | 上传中/等待上传文件计入空间 A 已用数量 |
| Space 内直接上传 | 上传中/等待上传文件计入该 Space 已用数量 |
| 展示一致性 | 添加到空间弹窗 + Space 列表卡片均展示含上传中/等待上传的数量 |
| 包含上传中/等待上传导致空间满 | 该空间置灰不可选，容量数值标红；Space 内添加来源弹出"存储空间已满" |

### 容量返还

| 场景 | 容量行为 |
|------|---------|
| 等待上传 → 上传失败 | 空间已用数量返还 |
| 上传中 → 上传失败 | 空间已用数量返还 |
| 上传中 → 点击取消 | 空间已用数量返还 |
| 等待上传 → 上传成功 | 服务端重新扣除容量，客户端释放（不重复扣除） |
| 上传中 → 上传成功 | 同上 |

### 跨设备容量同步

| 规则 | 说明 |
|------|------|
| B 设备查看空间 A 数量 | **不包含** A 设备上传中/等待上传文件（客户端本地状态不多端同步） |

### 特殊记录的容量占用

| 记录状态 | 是否占用容量 | 释放条件 |
|---------|------------|---------|
| 解析失败记录 | ✅ 占用 | 删除后释放 |
| 置灰快捷方式（原文件已删到回收站） | ✅ 依然占用 | 原文件从回收站**彻底删除**后释放 |

## 九、校验优先级

上传文件时，多种校验按以下优先级执行（先校验先拦截）：

| 优先级 | 校验项 | 校验方 |
|--------|--------|--------|
| 1 | 套餐信息获取（网络异常） | 客户端 |
| 2 | 音频时长超限 | 客户端 |
| 3 | 文件大小超限 | 客户端 |
| 4 | 转写时长不足（Basic/Pro） | 客户端 |
| 5 | 空间容量已满 | 客户端 + 服务端兜底 |

> 时长与大小同时超限时仅显示时长超限弹窗（时长校验优先于大小校验）。

---

## 十、多端并发校验（服务端兜底）

| 场景 | 行为 |
|------|------|
| 客户端校验空间未满 → 其他端导致空间满 → 当前设备提交 | 服务端校验拦截，上传失败 |
| 客户端校验空间数未满 → 其他端新建空间导致满 → 当前设备新建 | 服务端校验拦截，提示"空间数量已达上限" |
| 客户端校验转写时长充足 → 其他端消耗时长 → 等待上传变上传中 | 服务端校验失败，上传失败（无提示） |
| 客户端校验模板数未满 → 其他端创建导致满 → 当前设备保存 | 服务端校验拦截，提示"自定义模板达到上限" |

## 十一、测试关注点

- 两个入口上传音频，配置项差异（记录列表有「添加到空间」，Space 内无）
- 文档上传无弹窗，直接进入队列
- 时长+大小同时超限时，只弹时长超限弹窗（校验优先级）
- 解析失败删除不进回收站（与正常删除不同）
- **上传队列全局限制**（6 条含 Record 和所有 Space）；队列满 Toast 提示
- 文档不消耗转写时长（套餐无差异）
- Basic 用户上传 1h 边界、Pro/Ultra 用户上传 4h 边界
- **耳机上云队列**与文件上传队列互不影响（1 个 vs 3 个）
- 上云失败三种重试触发：点击图标 / 进入详情 / 切换 Tab
- **网络切换**：耳机上云双向切换不中断；文件上传 WiFi→4G 从头重传
- 不支持断点续传
- 上传进度弹窗 Record 和 Space 共享内容
- 上传失败记录持久化（重启 App 不消失）
- 切换账户：上传任务取消，重登后显示上传失败状态
- **转写时长消耗提示**：Basic/Pro 均显示（不分开关状态）；Ultra 不显示
- **空间容量占用**：上传中/等待上传计入已用数量；失败后返还；跨设备不同步
- **置灰快捷方式**依然占用空间容量，彻底删除原文件后才释放
- **多端并发校验**：客户端通过 → 其他端变更 → 服务端兜底拦截（空间容量/数量/模板/时长）
- **套餐过期时上传**：上传中继续完成；等待上传变为上传失败（无提示）
