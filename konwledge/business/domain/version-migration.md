# 新老版本数据同步规则（viaim）

> 版本：V2.0.0 | 最后更新：2026-03

---

## 一、分组 → 空间迁移规则

老版本"分组"概念在 V2.0.0 中升级为"空间（Space）"。

| 迁移规则 | 说明 |
|----------|------|
| 分组 → 空间 | 每个老版本分组自动创建同名空间 |
| 分组内记录 → 快捷方式 | 分组内记录在对应空间内创建快捷方式，Record 列表保留源记录 |
| 空分组（无记录） | 创建同名空间，空间内记录为空 |
| 无分组用户 | 不创建同步空间，空间列表仅有示例空间（1 个） |
| 多个分组 | 空间数量与原分组数量一致 |
| 分组排列顺序 | 同步空间保持老版本中调整后的排列顺序 |

---

## 二、示例空间与示例文件

| 规则 | 说明 |
|------|------|
| 自动创建 | 所有升级用户自动创建示例空间，示例空间内同步创建示例文件 |
| 空间总数公式 | 空间总数 = 老版本分组数 + 1（示例空间） |
| 示例文件计入容量 | 示例文件数计入单个空间记录数限制 |
| Record 列表 | 自动创建示例文件《Welcome to viaim》.mp3，创建时间为首次登录时间 |
| 演示 Space | 《Welcome to viaim》空间，含示例 .mp3（快捷方式）和 .pdf（文档） |
| 支持编辑和删除 | 示例文件/演示 Space 支持编辑和删除操作 |
| 删除后不重建 | 删除后再次登录不会重新创建 |
| 再次登录展示位置 | 已有示例文件再次登录 → 展示在第一次生成的时间位置 |
| 老用户同步页面 | 老版本用户首次登录 2.0 → 显示"正在同步记录"页面，同步成功后展示记录 |
| 升级安装 | 登录状态下升级安装 App → 不显示同步页面，直接展示记录列表 |

## 三、「分组已升级为空间」弹窗

| 规则 | 说明 |
|------|------|
| 触发条件 | 老用户首次登录新版本，首次进入空间 Tab |
| 新注册用户 | 不弹出 |
| 再次进入 | 同次登录再次进入不弹出 |
| 未操作 10s | 弹窗自动消失 |
| 关闭方式 | 点击"知道了" / X / 按住下拉 / 弹窗外 |
| 卸载重装 | 再次弹出 |

## 四、祖父条款详细规则

> 各套餐权益上限值以 `subscription.md` 第一节为准。以下为迁移场景的临界值计算。

升级后已有数量超出新套餐上限时：**数量保留，不可新建/新增，删至上限以下才恢复新增能力。**

### 4.1 空间数量

| 套餐 | 上限 | 临界值（分组数 + 1 示例 = 上限） | 超限行为 |
|------|------|------|------|
| Basic | 10 | 分组 = 9 → 总数 10 = 上限 | = 上限时不可新建；< 上限可新建 |
| Pro | 20 | 分组 = 19 → 总数 20 = 上限 | 同上 |
| Ultra | 40 | 分组 = 39 → 总数 40 = 上限 | 同上 |

**恢复规则**：删除后总数 < 上限 → 恢复新建能力；总数 = 上限 → 仍不可新建。

### 4.2 单空间文件数

| 套餐 | 上限 | 超限行为 |
|------|------|------|
| Basic | 10 | 分组内记录 > 10 → 快捷方式全部保留，不可新增 |
| Pro / Ultra | 50 | 分组内记录 > 50 → 快捷方式全部保留，不可新增 |

**恢复规则**：删至 < 上限 → 恢复新增能力；= 上限 → 仍不可新增。

## 五、Record 列表与空间关系

| 规则 | 说明 |
|------|------|
| Record 列表 | 展示所有源记录，不按空间区分 |
| 记录总数 | 升级前后 Record 列表记录总数一致 |
| 筛选"已加入空间" | 显示同步过来的已加入空间的记录 |
| 筛选"未加入空间" | 显示未加入任何分组/空间的记录 |

---

## 六、迁移数据的删除行为

| 操作 | 行为 |
|------|------|
| 删除空间内快捷方式 | 源记录不受影响，Record 列表仍显示 |
| 删除 Record 源记录 | 空间内对应快捷方式置灰（不删除） |
| 源记录从回收站恢复 | 空间内快捷方式恢复正常 |
| 删除空间 | 源记录不受影响，回收站展示该空间下所有快捷方式 |
| 置灰快捷方式 | 显示已删除/不可用状态标识，点击提示"原文件已被删除"，不可进入详情页 |

---

## 七、Space 列表排序规则

| 来源 | 排序规则 |
|------|----------|
| 老版本同步空间 | 保持原分组设置的排列顺序 |
| V2.0 新建空间 | 按创建时间倒序排列 |
| 演示 Space | 按创建时间参与正常排序 |
| 混合排列 | 老版本同步空间在后，新建空间在前（因新建时间更晚） |

---

## 八、测试关注点

- 空间总数 = 分组数 + 1（示例空间），不要遗漏示例空间
- 各套餐临界值：Basic 分组 9/10/11、Pro 19/20/21、Ultra 39/40/41
- 祖父条款恢复条件：< 上限才恢复，= 上限仍不可新建
- 无分组老用户升级后空间列表仅有示例空间（1 个），不为空
- "分组已升级为空间"弹窗仅老用户首次进入触发，卸载重装会再次触发
- Record 列表记录总数升级前后一致
- 删除源记录后空间内快捷方式置灰而非删除
- 迁移数据的筛选功能（已加入/未加入空间）正确性
